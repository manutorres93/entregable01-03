MySQL: VIEWS y Procedimientos Almacenados

Views

1. Una vista es una tabla virtual que no necesitamos crear una tabla adicional y solo se guarda la consulta SELECT. No almacena datos por si mismo

2. Se puede usar como medida de seguridad para limitar lo que un usuario puede ver, se le da acceso a las vistas y no a las tablas

3. Oculta la complejidad de las consultas para que sea más fácil su lectura

EJEMPLOS VIEWS

CREATE VIEW v_mayores_de_25 AS SELECT * FROM users WHERE edad > 25;

CREATE VIEW v_invitacion_concierto_reggaeton AS SELECT * FROM users WHERE musica = "reggaeton"

CREATE VIEW v_nombres_completos AS SELECT nombres, apellidos FROM users

CREATE VIEW v_usuarios_por_pais AS SELECT pais, COUNT(*) AS cantidad_por_pais FROM users GROUP BY pais;

CREATE VIEW v_orden_por_edad AS SELECT * FROM users ORDER BY edad DESC;

Fuente:_https://dev.mysql.com/doc/refman/8.0/en/view-syntax.html

Procedimientos almacenados

Los procedimientos almacenados son conjuntos de instrucciones SQL que se almacenan en el servidor de la base de datos bajo un nombre particular y pueden ser ejecutados por una aplicación o un cliente. Ayudan a encapsular lógica de negocio y facilitan la reutilización de código. Pueden recibir parámetros. Son bloques de código SQL que se pueden reutilizar. Se pueden usar para realizar tareas complejas, como:

  • Validar datos
  • Realizar cálculos complejos
  • Ejecutar múltiples consultas
  • Controlar el flujo de la lógica

EJEMPLOS PROCEDIMIENTOS ALMACENADOS

CREATE PROCEDURE insertar_persona( IN nombre VARCHAR(255), IN apellido VARCHAR(255), IN correo VARCHAR(255), IN edad INT, IN genero VARCHAR(255), IN pais VARCHAR(255), IN musica VARCHAR(255)) BEGIN INSERT INTO users (nombres, apellidos, correo, edad, genero, pais, musica) VALUES (nombre, apellido, correo, edad, genero, pais, musica); END // DELIMITER ;

DELIMITER // CREATE PROCEDURE actualizar_usuario( IN usuario_id INT, IN nombres VARCHAR(255), IN apellidos VARCHAR(255), IN correo VARCHAR(255), IN edad INT, IN genero VARCHAR(255), IN pais VARCHAR(255), IN musica VARCHAR(255) ) BEGIN UPDATE users SET nombres = nombres, apellidos = apellidos, correo = correo, edad = edad, genero = genero, pais = pais, musica = musica WHERE id = usuario_id; END // DELIMITER ;

DELIMITER // CREATE PROCEDURE actualizar_musica(IN persona_id INT, IN nueva_musica VARCHAR(255)) BEGIN UPDATE users SET musica = nueva_musica WHERE id = persona_id; END // DELIMITER ;

DELIMITER // CREATE PROCEDURE actualizar_nombres(IN persona_id INT, IN nuevo_nombre VARCHAR(255)) BEGIN UPDATE users SET nombres = nuevo_nombre WHERE id = persona_id; END // DELIMITER

DELIMITER // CREATE PROCEDURE eliminar_usuario( IN user_id INT ) BEGIN DELETE FROM users WHERE id = user_id; END // DELIMITER ;

Diferencia entre SQL y NoSQL

SQL (Structured Query Language) es un lenguaje de consulta para bases de datos relacionales, mientras que NoSQL se refiere a un enfoque más flexible y escalable para el almacenamiento y recuperación de datos, que no sigue un modelo de base de datos relacional.

Base de datos relacional:

Los datos se almacenan en tablas con relaciones entre ellas. Estructurada: Los datos se organizan en filas y columnas con tipos de datos específicos. Lenguaje de consulta declarativo: Se describe qué se quiere obtener, no cómo obtenerlo. Adecuado para: Transacciones, análisis de datos, aplicaciones con relaciones complejas.

NoSQL

Base de datos no relacional: Los datos se pueden almacenar en diferentes formatos, como documentos, columnas o claves-valor. No estructurada: Los datos pueden tener diferentes estructuras y no necesitan tener un tipo de dato específico. Lenguaje de consulta imperativo: Se describe cómo obtener los datos. Adecuado para: Aplicaciones con grandes volúmenes de datos, aplicaciones con estructuras de datos flexibles, aplicaciones que necesitan escalabilidad horizontal.

MongoDB

MongoDB es una base de datos NoSQL orientada a documentos. Es una base de datos distribuida y escalable que se puede usar para almacenar grandes volúmenes de datos.
